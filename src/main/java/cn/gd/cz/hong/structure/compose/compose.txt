组合模式

通过树的形式组合对象   用于表示部分与整体

分为俩种组合模式 分别是
透明式以及安全式

这次我使用的是安全式的
也就是叶子结点（没有后继结点的结点）只拥有必须的方法 而不包括操作非叶子结点的结点 的方法
所以这是安全

而透明式的叶子结点操作的方法也包括操作拥有子节点的结点
虽然这样确实可以完全透明的使用 任何结点操作都是一样
但是确实是不安全 毕竟叶子结点拥有了过多了不属于自身属性的方法
这样的叶子结点根本就没有体现出来他是一个叶子结点的觉悟

本实例中使用 公司 分公司 部门 做实例
本来打算将员工也包括进来
但是后来经过思考之后员工与这些树状结构根本就不能算是一个对象 也就是说属性 以及 行为大相径庭
所以 后来也就舍弃了将员工包括进来这个想法
