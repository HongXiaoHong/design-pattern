桥接模式 
bridge 

目的：
	实现抽象与实现的分离 从而达到解耦的效果
	
实现方式：
	通过组合代替继承
	
现实生活中的例子
厂家的角度出发
沐浴露
气味分类
有的人喜欢芳香型 有的人喜欢清爽型 有的人喜欢浓香型 有的人喜欢无香型

还有一个例子就是
生产不同形状的不同颜色的灯泡
如果使用继承的话  要多做好几种材料  不同形状的都做出不同颜色的出来

如果使用组合那么只需要做出形状 然后再上色就可以了

功效分类
有的人喜欢有生发功能的 有的人喜欢柔顺头发  有的人喜欢让头发亮丽 有的人想让头发乌黑

这也就要求厂家造就不同的产品
如果从继承的角度出发 这里需要16个类才能满足顾客的需求

如果从组合的角度来讲 要什么拿什么 则需要8个类就可以了


还有就是制作背包的时候

顾客喜欢的颜色以及种类 可以当作俩个维度
我们将这俩个维度进行抽象
不依赖实现
而是依赖抽象
